# Product Specification: Information Architecture

## Navigation Paradigm

The OS does not use a traditional SaaS left-hand sidebar with isolated "apps." It presents a unified production dashboard with the Visual Engine available as an overlay.

## The ERP Dashboard

The default view. A dense, rigorous interface where the studio manages its operations.

- **Command Center.** The aggregate state: "3 invoices overdue," "2 workflows in progress today," "1 intent awaiting review."
- **Financial Ledger.** All Financial Transactions — invoices, deposits, expenses, revenue — in one unified view.
- **Operations.** Calendar, resource availability, staff assignments, active workflows.
- **People.** The complete record of every person — clients, staff, vendors — and their history with the Organization.

The user does not "launch apps." They filter and navigate facets of the same underlying Production Engine.

## The Visual Engine Overlay

Whenever the studio needs to *present* data rather than *manage* it, they summon the Visual Engine.

- Looking at a service definition → "Design Booking Page" → Visual Engine opens.
- Looking at a completed workflow → "Design Gallery" → Visual Engine opens.
- Looking at a financial transaction → "Design Invoice Template" → Visual Engine opens.

Same tool. Same canvas. Different data context.

## Navigation Flow

1. Log in. Land on the Command Center.
2. Manage operations, review intents, track workflows, reconcile finances.
3. When presentation is needed, summon the Visual Engine over any data object.
